# **Лабораторная работа №1**
Данная работа посвящена изучению архитектуры программно-определяемого радио **ADALM-Pluto SDR** и созданию FM-радиоприёмника с использованием фреймворка **GNU Radio**. В процессе выполнения практики был реализован полный цикл построения приёмного тракта — от получения сигнала на антенне до его демодуляции и вывода на аудиоустройство.

## **Цель работы**
Ознакомиться с архитектурой SDR-устройства ADALM Pluto и построить FM-радиоприёмник, работающий в реальном времени, с использованием GNU Radio.

## **Краткие теоретические сведения**
Программно-определяемое радио (**SDR**) — это технология, позволяющая выполнять основные функции радиоприёмника программными методами. Такие этапы, как фильтрация, демодуляция и обработка спектра, осуществляются цифровой обработкой сигналов, что обеспечивает гибкость настройки и быстроту конфигурации устройства.

GNU Radio предоставляет визуальную среду **GNU Radio Companion (GRC)**, где радиотракт создаётся путём соединения функциональных блоков без написания кода.
## **Ход работы**
### **1. Установка GNU Radio**
GNU Radio был установлен через стандартный менеджер приложений Ubuntu. После установки была проверена работоспособность GNU Radio Companion.

### **2. Настройка источника сигнала (PlutoSDR Source)**
В блоке **PlutoSDR Source** были заданы:

* **IIO context URI** — IP-адрес устройства;
* **LO Frequency** — частота принимаемой FM-станции;
* **Sample Rate** — частота дискретизации сигнала;
* **RF Bandwidth** — ширина полосы пропускания;
* **Buffer Size** — размер буфера.

Этот блок обеспечил получение сигнала с устройства ADALM Pluto в реальном времени.

### **3. Визуализация спектра**
Блок **QT GUI Frequency Sink** позволил наблюдать спектр принимаемого сигнала и точнее настроиться на FM-станцию.

Настраивались параметры:

* FFT Size
* Center Frequency
* Bandwidth


### **4. Фильтрация сигнала**
Для удаления лишних спектральных составляющих использовался **Low Pass Filter**, где были выбраны:

* коэффициент усиления,
* частота среза,
* ширина переходной полосы,
* тип окна,
* decimation для приведения частоты к формату обработки.


### **5. Демодуляция FM-сигнала**
Блок **WBFM Receive** осуществлял демодуляцию широковещательного FM-сигнала.

Настроены параметры:

* **Quadrature Rate**,
* **Audio Decimation** для последующего вывода звука.


### **6. Вывод аудио**
Для прослушивания сигнала использовался блок **Audio Sink**, куда был подан демодулированный сигнал с заданной частотой дискретизации.


### **7. Дополнительная визуализация**
Блок **QT GUI Time Sink** применялся для отображения временного представления сигнала, что помогло оценить его форму и стабильность.


### **8. Управление параметрами**
Через **QT GUI Range** был добавлен удобный интерфейс-ползунок для динамического изменения частоты настройки (например, диапазон 88.7–110.1 МГц).


## **Итог**
Путём соединения всех перечисленных блоков была создана рабочая схема FM-радиоприёмника, отображающая спектр сигнала, временную область и воспроизводящая аудио в реальном времени.
